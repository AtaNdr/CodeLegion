Non-Negotiable Criteria
Five requirements for a stakeholder-grade automation system
Unattended
No human-in-loop for routine work. Agents act and submit PRs autonomously.
Cost-controlled
Pay only for work done. Hard caps per task. Stop-fleet in one click.
Auditable
Every action visible on GitHub — issue comments, PR diffs, cost summaries.
Native to workflow
GitHub Issues and PRs only. No new tools, no new portals, no context switching.
Secure by default
No broad credentials on workers. Branch protection enforced. Short-lived tokens only.
A solution missing any one of these is not deployable in a serious environment.

Risks & Mitigations
Known risks — each with a concrete mitigation in place
LLM produces incorrect code
›
Every PR is human-reviewed. Tests required per criterion. Branch protection prevents direct main pushes.
Unbounded API spend
›
Per-task budget caps (time, tokens, turns). Hard model caps. Stop-fleet single click. Cost visible per task.
Dashboard exposed publicly
›
Azure App Service Easy Auth available today. Built-in password gate in active design.
Single-instance failure
›
App Service auto-restart. Persistent disk survives restarts. Reconcile picks up within ~45 seconds.
LLM provider lock-in
›
Prompt-driven interface (Claude Code CLI) is portable to other providers with comparable capability.
